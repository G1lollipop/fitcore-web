# FitCore 网页端功能优化方案

## 一、问题分析

### 1.1 当前问题概述

| 问题 | 表现 |
|------|------|
| **模块边界模糊** | 今日概览、锻炼日志、饮食记录三者内容高度重叠 |
| **数据不互通** | 今日锻炼模块与训练计划模块数据割裂 |

### 1.2 当前模块功能对比

| 功能点 | 今日概览 | 饮食记录 | 锻炼日志 | 训练计划 |
|--------|----------|----------|----------|----------|
| 统计卡片 | ✅ | ✅ | ✅ | ❌ |
| 营养追踪 | ✅ | ✅ | ❌ | ❌ |
| 快速记录(饮/食/练) | ✅ | ❌ | ❌ | ❌ |
| 今日训练计划 | ✅ | ❌ | ❌ | ✅ |
| 训练记录列表 | ❌ | ❌ | ✅ | ❌ |
| 计划管理 | ❌ | ❌ | ❌ | ✅ |

## 二、优化方案

### 2.1 模块重构：清晰边界与差异化定位

#### 方案设计

```
┌─────────────────────────────────────────────────────────────────┐
│                        主菜单重构                                │
├─────────────────────────────────────────────────────────────────┤
│  今日概览        │  核心定位：数据仪表盘 + 快速记录入口          │
│  (Dashboard)     │  展示：当日关键指标 + 快速记录               │
│                  │  动作：点击跳转到对应详情页                  │
├──────────────────┼──────────────────────────────────────────────┤
│  饮食记录        │  核心定位：饮食管理全量视图                  │
│  (Nutrition)     │  展示：今日饮食 + 历史记录 + 营养分析        │
│                  │  动作：查看/编辑/删除饮食记录                │
├──────────────────┼──────────────────────────────────────────────┤
│  训练记录        │  核心定位：训练历史全量视图                  │
│  (Workout Log)   │  展示：训练日历 + 历史记录 + 统计图表        │
│                  │  动作：查看/分析历史训练                     │
├──────────────────┼──────────────────────────────────────────────┤
│  我的计划        │  核心定位：计划管理 + 今日执行                │
│  (My Plans)     │  展示：计划列表 + 详情 + 今日训练            │
│                  │  动作：创建/编辑/切换/执行计划                │
└──────────────────┴──────────────────────────────────────────────┘
```

#### 具体改动

| 模块 | 新名称 | 定位 | 包含内容 |
|------|--------|------|----------|
| 今日概览 | **仪表盘** | 当天数据概览 + 快速入口 | 关键指标卡片、快速记录区、本周活跃度 |
| 饮食记录 | **饮食中心** | 饮食数据管理 | 营养目标进度、饮食记录列表(按日期)、统计图表 |
| 锻炼日志 | **训练历史** | 训练数据管理 | 训练日历、记录列表(按日期)、统计图表 |
| 训练计划 | **我的计划** | 计划管理+执行 | 计划列表、详情查看、今日训练执行 |

### 2.2 联动优化：今日锻炼 ↔ 训练计划

#### 问题分析

| 问题点 | 现状 |
|--------|------|
| 数据来源不同 | 今日锻炼从 dashboard 获取，计划从 plans 获取 |
| 展示内容重复 | 今日训练计划同时出现在「今日概览」和「训练计划」模块 |
| 操作割裂 | 导入计划在今日概览，修改计划在训练计划 |

#### 联动优化设计

```
┌────────────────────────────────────────────────────────────────────┐
│                        联动流程优化                                 │
├────────────────────────────────────────────────────────────────────┤
│                                                                    │
│   【训练计划模块】                        【今日概览/仪表盘】       │
│   ┌─────────────┐                       ┌─────────────────┐        │
│   │ 计划列表    │  设置为当前计划 ──────▶│  显示今日训练   │        │
│   │             │                       │  (从计划读取)   │        │
│   └─────────────┘                       └─────────────────┘        │
│         │                                        │                 │
│         │  修改计划                              │  执行训练       │
│         ▼                                        ▼                 │
│   ┌─────────────┐                       ┌─────────────────┐        │
│   │ 编辑计划    │  ─────────────────────▶│  记录训练结果   │        │
│   │ (CRUD)     │      实时同步更新       │  (写入记录)    │        │
│   └─────────────┘                       └─────────────────┘        │
│                                                                    │
└────────────────────────────────────────────────────────────────────┘
```

#### 关键改动点

| 改动 | 说明 |
|------|------|
| 统一数据源 | 今日训练数据统一从 `workout_plans` 读取，不再单独存储 |
| 移除冗余字段 | `user_settings.current_plan_id` 保持作为当前计划唯一引用 |
| 实时联动 | 修改计划后，自动刷新「今日概览」中的训练展示 |
| 双向跳转 | 从今日训练可跳转到计划详情，从计划详情可开始今日训练 |

## 三、具体实施步骤

### 步骤 1：菜单导航重构

**文件**: `components/sidebar-nav.tsx`

```typescript
// 新的导航结构
const navItems = [
  { id: "dashboard", label: "仪表盘", icon: LayoutDashboard },
  { id: "nutrition", label: "饮食中心", icon: UtensilsCrossed },
  { id: "training", label: "训练历史", icon: Dumbbell },
  { id: "plans", label: "我的计划", icon: Calendar },
  { id: "knowledge", label: "知识库", icon: BookOpen },
]
```

### 步骤 2：今日概览(仪表盘)简化

**文件**: `app/page.tsx`

| 改动 | 说明 |
|------|------|
| 移除重复组件 | 移除独立的饮食记录/锻炼日志入口卡片 |
| 保留核心 | 统计卡片 + 本周活跃度 + 快速记录入口 |
| 链接跳转 | 快速记录点击后展开详情，或跳转至对应模块 |

### 步骤 3：饮食记录模块增强

**新增**: `components/nutrition-center.tsx`

| 功能 | 说明 |
|------|------|
| 日期选择器 | 选择查看不同日期的饮食记录 |
| 记录列表 | 按时间排序的饮食记录 |
| 营养图表 | 蛋白质/碳水/脂肪摄入可视化 |
| CRUD 操作 | 添加/编辑/删除记录 |

### 步骤 4：训练记录模块增强

**新增**: `components/training-history.tsx`

| 功能 | 说明 |
|------|------|
| 训练日历 | 可视化展示每月训练日期 |
| 历史列表 | 按日期分组的训练记录 |
| 统计图表 | 周/月训练时长、次数趋势 |
| 详情查看 | 点击查看单次训练详情 |

### 步骤 5：「我的计划」模块整合

**文件**: `components/my-plans.tsx` (重构自 workout-plans.tsx)

| 功能 | 说明 |
|------|------|
| 计划卡片 | 展示计划基本信息 + 训练日 |
| 今日训练 | 直接在计划页面展示今日训练内容 |
| 一键开始 | 点击开始今日训练 |
| 快速记录 | 记录完成后直接在此页面记录 |

### 步骤 6：数据联动优化

**核心逻辑**:

1. **统一数据获取**: 今日训练数据统一从 `getCurrentPlan` 获取
2. **移除重复代码**: 删除 `getTodayWorkout` (已在步骤5中整合)
3. **实时刷新**: 计划变更后通过 `revalidatePath` 刷新相关页面
