# 训练计划详情显示问题修复方案

## 问题定位

经过代码分析，发现以下问题导致训练日详情无法显示：

### 核心问题：`copyTemplateToUser` 函数未复制训练动作

**位置**: [plans.ts:301-382](d:\Projects\Vibe-coding\fitcore\app\actions\plans.ts#L301-L382)

**问题描述**:
当用户使用系统模板创建计划时，`copyTemplateToUser` 函数只复制了 `workout_days`（训练日），但**没有复制 `plan_exercises`（训练动作）**。

**代码证据**:
```typescript
// L344-372 - 只复制了 workout_days，没有复制 plan_exercises
const { data: templateDays, error: daysError } = await (supabase as any)
  .from('workout_days')
  .select('*')  // ❌ 只获取了 workout_days 的字段，没有获取 plan_exercises
  .eq('plan_id', templateId)
  .order('day_order');

if (templateDays && templateDays.length > 0) {
  const newDays = templateDays.map((day: any) => ({
    plan_id: newPlan.id,
    name: day.name,
    // ... 其他字段
    // ❌ 没有 plan_exercises
  }));

  const { error: insertDaysError } = await (supabase as any)
    .from('workout_days')
    .insert(newDays);
  // ❌ 没有复制 plan_exercises
}
```

### 次要问题：`PlanDetailModal` 的 `rest_days` 处理逻辑

**位置**: [workout-plans.tsx:635-668](d:\Projects\Vibe-coding\fitcore\components\workout-plans.tsx#L635-L668)

**问题描述**:
当计划有 `rest_days` 字段时，组件使用复杂的索引映射逻辑来匹配训练日。虽然逻辑基本正确，但存在以下潜在问题：

1. 如果 `workout_days` 数据不完整或顺序不对，会导致 `workoutDay` 为 `undefined`
2. 没有处理 `rest_days` 为空数组但 `hasRestDays` 判断错误的情况

---

## 修复方案

### 步骤 1: 修复 `copyTemplateToUser` 函数

**修改文件**: [plans.ts](d:\Projects\Vibe-coding\fitcore\app\actions\plans.ts)

**修改内容**:
1. 获取模板训练日时，同时获取关联的 `plan_exercises`
2. 复制训练日后，将 `plan_exercises` 也复制到新的训练日

**具体修改**:

```typescript
// 修改获取模板训练日的查询，添加 plan_exercises
const { data: templateDays, error: daysError } = await (supabase as any)
  .from('workout_days')
  .select(`
    *,
    plan_exercises (*)
  `)
  .eq('plan_id', templateId)
  .order('day_order');

// 复制训练日时，保存新旧 ID 映射
const dayIdMap = new Map<string, string>();

for (const day of templateDays) {
  const { data: newDay, error: insertDayError } = await (supabase as any)
    .from('workout_days')
    .insert({
      plan_id: newPlan.id,
      name: day.name,
      day_order: day.day_order,
      focus_muscles: day.focus_muscles,
      day_type: day.day_type,
      rest_day: day.rest_day,
      estimated_duration_minutes: day.estimated_duration_minutes,
    })
    .select()
    .single();

  if (newDay) {
    dayIdMap.set(day.id, newDay.id);
    
    // 复制 plan_exercises
    if (day.plan_exercises && day.plan_exercises.length > 0) {
      const exercisesToInsert = day.plan_exercises.map((pe: any) => ({
        day_id: newDay.id,
        exercise_id: pe.exercise_id,
        order_index: pe.order_index,
        target_sets: pe.target_sets,
        target_reps_min: pe.target_reps_min,
        target_reps_max: pe.target_reps_max,
        target_weight_kg: pe.target_weight_kg,
        rest_seconds: pe.rest_seconds,
        notes: pe.notes,
      }));

      await (supabase as any)
        .from('plan_exercises')
        .insert(exercisesToInsert);
    }
  }
}
```

### 步骤 4: 添加数据验证

**修改文件**: [workout-plans.tsx](d:\Projects\Vibe-coding\fitcore\components\workout-plans.tsx)

**修改内容**:
在 `displayDays` 计算中添加数据验证：

```typescript
const displayDays = hasRestDays ? (() => {
  const dayNames = ['周一', '周二', '周三', '周四', '周五', '周六', '周日'];
  return dayNames.map((name, index) => {
    const dayOrder = index + 1;
    const isRestDay = restDays.includes(dayOrder);
    
    if (isRestDay) {
      return { name, isRestDay: true, plan_exercises: null, estimated_duration_minutes: null };
    }

    const nonRestDayIndex = dayNames.slice(0, index).filter((_, i) => !restDays.includes(i + 1)).length;
    const workoutDay = workoutDays[nonRestDayIndex];
    
    // ✅ 添加数据验证
    if (!workoutDay) {
      console.warn(`Missing workoutDay for index ${index}, nonRestDayIndex ${nonRestDayIndex}`);
      return { name, isRestDay: false, plan_exercises: null, estimated_duration_minutes: null };
    }

    return {
      name: workoutDay.name,
      isRestDay: false,
      plan_exercises: workoutDay.plan_exercises,
      estimated_duration_minutes: workoutDay.estimated_duration_minutes,
    };
  });
})() : workoutDays.map((day: any) => ({
  name: day.name,
  isRestDay: day.rest_day || false,
  plan_exercises: day.plan_exercises,
  estimated_duration_minutes: day.estimated_duration_minutes,
}));
```
